/compact

Description

/compact instructs Codex CLI to compress the current conversation history into a shorter summary, freeing up context window tokens for longer coding sessions.

When a session accumulates many turns of dialogue, model responses and tool call outputs consume a large portion of the context window. Running /compact replaces verbose history with a condensed summary while retaining key decisions, file paths, and in-progress task context.

Syntax
shell 
# Basic usage — compact with default summary
> /compact

# With custom instructions for what to preserve
> /compact Focus on the authentication refactor. Keep all file paths.
Parameters
Parameter Type Description
[instructions] string, optional Natural language hints passed to the model about what context to preserve in the summary.
Examples

After a long debugging session — reset context before new feature work:

shell session 
> /compact Keep the bug in auth.ts we just fixed and the test structure. Forget the failed attempts.

✓ Conversation compacted (84 → 12 turns)
  Preserved: auth.ts fix, test file structure, current task context
  Freed: ~18 400 tokens

Compact before switching to a new subtask in the same session:

shell session 
> /compact Summarise the database migration work. I am now working on the API layer.

✓ Conversation compacted (61 → 9 turns)
  Freed: ~14 200 tokens
Notes
  • Compaction uses the model itself to write the summary — quality depends on how well you phrase the optional instructions. Be specific about tasks in progress.
  • The original conversation history is discarded after compaction — there is no undo. If you need the full context, use resume to start a new session instead.
  • Works best after a clearly defined phase has completed (e.g. after a bug fix, before switching features).
Caveats
  • The summary is non-deterministic — two identical invocations may produce slightly different summaries depending on the model's sampling.
  • Tool call outputs (large file reads, long command output) may be summarised aggressively. If a specific shell output is critical, quote it explicitly in your instructions.
